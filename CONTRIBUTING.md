# Contributing to Photonix

感谢您有兴趣为 Photonix 做出贡献！作为一个开源项目，我们非常欢迎社区的 Bug 反馈、功能建议及代码贡献。

## 🚀 如何开始

### 提交 Bug 反馈
如果发现程序异常，请在 GitHub Issues 中提交。请包含：
- **版本信息**：您的浏览器版本及 Docker 镜像版本。
- **重现步骤**：详细描述如何触发该 Bug。
- **错误日志**：通过 `docker compose logs` 获取的相关后端报错信息。
- **控制台截图**：对于前端问题，请附上浏览器开发者工具控制台的错误快照。

### 提交功能建议 (Feature Request)
我们欢迎任何能提升用户体验的想法。请描述您的应用场景，以及该功能对其他用户的潜在帮助。

---

## 🛠️ 开发指南

### 代码规范
- **后端 (Node.js)**: 遵循 JavaScript Standard Style，并确保所有新接口均配合对应的 Joi schema 进行验证。
- **前端 (Vanilla JS)**: 维持 SPA 设计风格，避免引入重型框架。UI 使用 Tailwind CSS，保持响应式兼容。
- **文档**: 使用 Markdown 编写，涉及专业技术名词时建议中英文结合。

### Git 流程
1. **Fork** 本仓库并创建您的特性分支 (Feature Branch)。
2. **Commit** 信息应当清晰且具有描述性（建议使用中文，格式如 `fix: 修复某某 Bug` 或 `feat: 新增某某功能`）。
3. **Draft PR**: 欢迎在工作尚未完全完成时提交 Draft Pull Request 以便获取早期反馈。

### 本地测试
在提交 PR 之前，请确保：
- 代码没有 Lint 错误。
- 如果修改了 API 逻辑，请完成基本的功能验证。
- 考虑到多种分辨率（移动端与桌面端）的兼容性。
